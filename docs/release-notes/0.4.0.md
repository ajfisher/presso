# Presso 0.4.0 Draft Release Notes

Presso 0.4.0 adds slide builds for Markdown-authored decks while keeping
exported reading and print formats fully built by default.

## Highlights

- Adds `:::fragment` slide-build authoring for click-through reveals.
- Reveals list fragments one direct list item at a time.
- Reveals mixed heading/list fragments in source order, so column titles can
  build before their bullets.
- Supports nested fragments inside column layouts for left-to-right build
  sequences.
- Synchronizes build state across the main deck, presenter view, and phone
  controller.
- Keeps static presentation HTML interactive while rendering print, PDF, notes,
  transcript, and manifest outputs in the final built state.

## Authoring

Use a fragment block around the material that should build:

```markdown
## Heading stays visible

:::fragment
- First point
- Second point
- Third point
:::
```

The heading appears immediately. Each `Next` reveals one bullet before the deck
advances to the next slide.

For non-list content, each top-level block becomes one build step:

```markdown
:::fragment
First paragraph.

> Second block.
:::
```

## Runtime

Presentation routes track a slide index and build step. `Next` advances through
builds first; `Back` reverses builds first, then returns to the previous slide
fully built. Hash URLs continue to support `#/3` and now also support
`#/3/2` for a specific build step.

## Export Compatibility

PDF, print, notes, transcript, and manifest outputs render slide content as if
all build steps have completed. Public static deck and embed routes remain
interactive so published slide decks can still be presented from static hosting.

## Expected Version

This release should be `0.4.0`. Release Please is configured with
`bump-minor-pre-major: true`, so the `feat:` commits in this slice should move
all linked Presso packages from `0.3.x` to `0.4.0`.
