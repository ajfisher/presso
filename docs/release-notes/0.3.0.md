# Presso 0.3.0 Draft Release Notes

Presso 0.3.0 is the migration-ready authoring release. It keeps the framework
lightweight and CSS-first, while adding the primitives needed to bring recent
Reveal talks across without inheriting Reveal's theme or plugin model.

## Highlights

- Adds structured slide background metadata with image, colour, fit, position,
  repeat, and opt-in overlay support.
- Renders explicit background hooks and CSS custom properties so themes no
  longer need to sniff inline styles.
- Adds the `image-title` layout for full-bleed background images with title
  panels.
- Adds nested `:::columns` / `:::column` directives for explicit two-column
  authoring.
- Adds the first minimal `presso migrate reveal <source> <target>` command.
- Documents the CSS-first theme authoring contract for deck-local themes.

## Authoring Changes

Backgrounds can now be written as shorthand or structured metadata:

```yaml
background: ./assets/images/lunar_lander.png
```

```yaml
background:
  image: ./assets/images/lunar_lander.png
  color: "#10131a"
  fit: cover
  position: center
  repeat: no-repeat
  overlay:
    type: scrim
    direction: left
    strength: 0.35
```

Image overlays now default to `none`. Decks that want a dark scrim should opt in
with `background.overlay` or theme-specific CSS.

Preferred column authoring is now explicit:

```markdown
:::columns
:::column
![Illustration](./assets/images/jfk.png)
:::

:::column
> Quote or body content.
:::
:::
```

Older simple `:::columns ... :::` wrappers continue to render, so existing
decks do not need immediate edits.

## Runtime And Theme Changes

Slides now expose `data-background="image"`, `data-background="color"`, or
`data-background="image color"` alongside background CSS custom properties:

- `--presso-bg-image`
- `--presso-bg-color`
- `--presso-bg-fit`
- `--presso-bg-position`
- `--presso-bg-repeat`
- `--presso-bg-overlay`

Themes should style deck-specific presentation treatment in the deck's
configured `theme.css`, using `data-layout`, `data-background`, directive
classes, and custom properties. The new theme authoring guide documents the
stable hooks.

## Migration Changes

`presso migrate reveal <source> <target>` now provides a practical bootstrap for
recent AJ-style Reveal decks:

- Discovers `src/slides.md`.
- Copies `src/images` to `assets/images`.
- Copies `src/static` to `public/static`.
- Splits Reveal slides into numbered Presso folder-mode slides.
- Converts `Notes:` blocks to `:::notes`.
- Maps common slide-level `class`, `data-background`, and `data-timing` values
  to Presso frontmatter.
- Converts obvious `twocolumn` wrappers to nested Presso column directives.
- Writes `MIGRATION.md` with unsupported `.element` comments and ambiguous
  manual cleanup items.

This is deliberately not a Reveal compatibility layer. Reveal themes, plugins,
and broad `.element` mappings remain manual cleanup.

## Export And Safety Changes

- Public `deck.json` manifests now carry structured background metadata.
- Transcript rendering understands structured background images and strips
  nested directive wrappers.
- Deck compilation rejects duplicate slide IDs with source context.
- Static and browser smoke coverage now includes the expanded example deck,
  image-title slides, logos, columns, and nested-route asset loading.

## Compatibility

- Existing `background: ./asset.png` shorthand remains supported.
- CSS colour shorthand is supported for background colours without Presso
  hard-coding the browser's named-colour registry.
- `backgroundFit` remains a legacy shorthand for image sizing.
- Existing simple `:::columns` directives remain supported.
- Raw HTML remains enabled by default.

## Expected Version

This release should be `0.3.0`. Release Please is configured with
`bump-minor-pre-major: true`, so the `feat:` commits in this slice should move
all linked Presso packages from `0.2.0` to `0.3.0`.
